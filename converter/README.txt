=== БЫСТРЫЙ СТАРТ: ПОЛНЫЙ ЦИКЛ (генерация + публикация) ===

  publish.bat — одним кликом: конвертирует Excel и публикует на сайт.

Перед первым запуском настройте SFTP (см. раздел «Настройка деплоя» ниже).


=== КАК ОБНОВИТЬ КАТАЛОГ (только генерация, без публикации) ===

1. Положите Excel-файл в папку  input/
   (старый файл можно удалить или оставить — используется последний)

2. Если изменились фото — скопируйте их в папку  images/

3. Дважды кликните  run.bat  — конвертер запустится

4. Смотрите статус:
   ✓ УСПЕШНО           — всё в порядке, файлы готовы в out/
   ✓ УСПЕШНО с предупреждениями — проверьте warnings в logs/
   ✗ ОШИБКА            — исправьте Excel и запустите снова

5. После run.bat запустите  deploy.bat  — файлы уйдут на сайт автоматически.


=== ПАПОЧНАЯ СТРУКТУРА ===

converter/
├─ input/                 ← кладите .xlsx сюда
├─ images/                ← исходные фото товаров (.webp/.jpg/.png)
├─ out/                   ← готовый пакет для сайта
│  ├─ products.js
│  ├─ products.generated.json  (для проверки/аудита)
│  ├─ summary.txt              (итоговый отчёт)
│  ├─ missing_photos.txt       (если фото не найдены)
│  └─ assets/img/products/     (скопированные фото)
├─ logs/                  ← журнал конвертации и деплоев
├─ backup/                ← локальные бэкапы (конвертер + серверные файлы)
│  ├─ 20260416_120000/    ← бэкап генерации
│  └─ server_20260416_120500/ ← бэкап файлов с сервера перед деплоем
├─ config/
│  ├─ settings.json       ← настройки конвертера
│  ├─ mapping.json        ← маппинг колонок Excel
│  ├─ dictionaries.json   ← словари значений
│  ├─ deploy.example.json ← шаблон настроек деплоя
│  └─ deploy.json         ← ваши настройки SFTP (не попадает в git!)
├─ run.bat                ← только генерация из Excel
├─ run_dryrun.bat         ← проверка без записи файлов
├─ publish.bat            ← генерация + деплой (полный цикл)
├─ deploy.bat             ← только деплой (если уже сгенерировано)
├─ rollback.bat           ← откат к предыдущей версии
└─ preview.bat            ← предпросмотр изменений без деплоя


=== СТРУКТУРА EXCEL ===

Обязательные колонки:
  id            — числовой ID (уникальный)
  sku           — артикул (уникальный, без пробелов)
  brand_name    — название бренда
  model         — модель (только номер модели, напр. ELB-I07PN)
  catalog_group — группа: inv / onoff / accessory / truba / krepezh
  price         — цена в рублях (число)
  stock_status  — статус: in_stock / days_1_2 / days_3_5 / order_7
  stock_label   — подпись статуса (напр. "В наличии")
  description_short — краткое описание
  photo         — имя файла фото (напр. ultima-elysium.webp)

Колонка active:
  1  — включить товар в каталог
  0  — не включать (строка будет пропущена)

modal_benefits — преимущества через разделитель |
  Пример: Компрессор GREE|Wi-Fi Ready|Режим ECO|2 года


=== НАСТРОЙКА ДЕПЛОЯ (первый раз) ===

1. Откройте папку  config/
2. Скопируйте  deploy.example.json  →  deploy.json
3. Откройте deploy.json и заполните:
   - host     — адрес хостинга (напр. splithub.ru или ftp.splithub.ru)
   - port     — порт SFTP (обычно 22)
   - username — логин SFTP
   - password — пароль SFTP  (или key_path — путь к SSH-ключу)
   - remote_path — полный путь к папке сайта на сервере
                   (напр. /home/username/public_html)

Файл deploy.json не попадает в git — пароль в безопасности.


=== РЕЖИМЫ ДЕПЛОЯ ===

publish.bat   — генерация Excel + деплой (один клик, полный цикл)
deploy.bat    — только деплой уже сгенерированного каталога
preview.bat   — показать изменения по сравнению с последним деплоем
rollback.bat  — откатить сайт к предыдущей версии каталога

Для деплоя только изображений (без products.js):
  Запустите в консоли: python deploy.py --images-only

Для проверки без реальной загрузки:
  Запустите в консоли: python deploy.py --dry-run


=== КАК ОТКАТИТЬСЯ НА ПРЕДЫДУЩУЮ ВЕРСИЮ ===

Быстрый способ: дважды кликните  rollback.bat
Скрипт найдёт последний успешный деплой и восстановит файлы на сервере.

Ручной способ:
  В папке backup/server_ДАТА/ лежат products.js и products.generated.json,
  сохранённые с сервера перед последним деплоем.
  Скопируйте нужный products.js обратно в out/ и запустите deploy.bat.


=== НАСТРОЙКИ (config/settings.json) ===

missing_photo_mode:
  "warn"  — фото не найдено = предупреждение, товар попадает в out/
  "error" — фото не найдено = ошибка, товар не попадает в out/

dry_run:
  true  — режим проверки, файлы не записываются
  false — обычный режим

valid_stock_statuses — список допустимых статусов наличия
valid_groups         — список допустимых групп товаров


=== ИЗМЕНИТЬ МАППИНГ КОЛОНОК ===

Если переименовали колонку в Excel — откройте config/mapping.json
Левая сторона — имя колонки в Excel, правая — поле в products.js
Пример: если "brand_name" переименовали в "бренд":
  "бренд": "brand",


=== ЗАВИСИМОСТИ ===

Требуется: Python 3.8+  (python.org)
Библиотеки (устанавливаются автоматически):
  openpyxl — чтение Excel (для конвертера)
  paramiko  — SFTP/SSH (для деплоя)


=== РЕЖИМ DRY-RUN (проверка) ===

run_dryrun.bat — запускает конвертер в режиме проверки:
  - Читает Excel, валидирует данные
  - Показывает ошибки и предупреждения
  - НЕ записывает файлы в out/ и не делает backup
  Используйте перед реальным обновлением.
